# 짐 화면 구조

> UI 복제 금지. **흐름·자리만** 빌린다.  
> 제품: `docs/dojo.md` · 수익: `docs/monetization.md`

## 최상위 구조 (현재)

첫 화면은 **풀맵**이다.

| 자리 | 내용 |
|------|------|
| 상단 검색 | 지역 검색 + 오른쪽 **체육관 · 라이벌** 칩 · 아래 지역 프리셋 |
| 왼쪽 레일 | **찜 · 문의 · 제보 · 관장** (필요 시 지도로 복귀) |
| 하단 시트 | 검색 목록(썸네일) · 핀 선택 시 **사진+정보 카드** · 상세(커버·소개) |

모임은 UI에서 내렸지만 코드·데이터는 유지한다. 메시지 알림 확장은 나중.

## 참고 앱에서 가져온 것

| 앱 | 빌린 구조 | 안 가져온 것 |
|----|-----------|--------------|
| **당근** 비즈프로필 | 찾기 → **가게 홈**(커버·소개·가격) → 문의·채팅 | 단골·쿠폰·후기 피드 |
| **플랩** | 장소/매치 카드 → **상세** → 신청(행동 1개) | 소셜 매치·레벨·구장 예약 풀스택 |
| **Boxing Network** | 짐/이벤트 **허브** + 문의·매칭은 다음 행동 | 팀 가입·bout sheet·코치 워크플로 |
| **더코너류** | 사용자가 **내 체육관**을 프로필에 연결 | 지금 전역 커뮤니티 피드 |

## 관 상세 = 가게 홈 (잠금)

한 장면: **커버 사진 · 관 이름 · 소개 · 위치 · 이용권 · 문의하기(또는 받은 문의)**.  
상세는 하단 시트 full에서 연다. 커뮤니티는 한 줄 자리만.

코드: `GymDetailPanel.jsx`

## 체육관 지도 — 보조 행동

| 행동 | 흐름 |
|------|------|
| **내 체육관으로 등록** | 지도 핀 → 상세 → 프로필 소속 저장 |
| **새 체육관 등록** | 없는 장소 제보 → 이름·주소·좌표 → 승인 대기 |
| **내 문의** | 보낸 문의 → 미리보기 → 대화 |
| **관장 입점 관리** | 입점 신청·수정 · 승인 상태 · 받은 문의 |

SQL: `dojo_inquiry_chat.sql` + `dojo_inquiry_chat_inbox.sql`  
안 읽음 확장·푸시·Realtime은 나중.

연락처는 공개하지 않는다. 답장은 앱 대화가 기본.

## 테마

- **라이트:** 흰 무대 · 잉크 텍스트 · 버건디는 CTA만. 스포츠/필름 분위기 없음.
- **다크:** 오일 블랙 + 버건디 틴트 (기존 브랜드감).
