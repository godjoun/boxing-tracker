| 탭 | 무엇 | 왜 |
|----|------|-----|
| **체육관** | 지도 · 상세 · 문의 · 내 체육관 등록 | 첫 진입의 주 무대 |
| **라이벌 찾기** | 공개 카드 · 상호 관심 매칭 · 1:1 대화 | 지역·시간 조건 기반 |
| **찜** | 저장한 체육관 카드 | 다시 찾을 장소 |

- 모임 UI는 보류. 기존 기능·데이터는 삭제하지 않음
- 연락처 공개 안 함
- 시드 카드 = 예시 (모임·라이벌 예시는 노출하지 않음)
- **체육관 검색** — 지도·OSM + **좌표 있는 입점(approved)** · 지역 밖·좌표 없는 입점 카드는 숨김
- **모임 일정·신청** — Supabase 연결 시 폰끼리 동기화 (`supabase/dojo_exchange.sql`) · 미연결이면 로컬
- **체육관 문의** — 리드 저장 · **받은/보낸 문의** · 대화하기 · `dojo_inquiry_chat.sql`
- **내 체육관** — 지도 결과에서 소속 저장 / 없는 장소는 사용자 제보
- **관장 입점** — 체육관 화면의 관장 관리 · `approved`면 지도에 입점 핀 · 승인은 Table Editor
- **관 상세** — 목록 → **가게 한 장면** (커버·소개·가격·문의) · 커뮤니티는 한 줄 자리만
- **광고/추천** — 목록 카드 끼워넣기 안 함 · 나중에 **지역 배너** (`is_featured`)

- **모임 채팅** — 신청자↔주최자 DM (`supabase/dojo_chat.sql`) · 실시간 없음(새로고침) · 예시 일정 제외
- **라이벌 찾기 v1** — 실제 공개 카드와 관심 요청 (`supabase/dojo_sparring_v1.sql`) · 연락처 공개 없음
- **라이벌 채팅 v2** — 서로 관심을 보낸 뒤에만 1:1 대화 (`supabase/dojo_sparring_chat.sql`) · 4초 폴링 · 푸시 없음

## 운영 메모

토스가 부산에서 시작한 것처럼 **군산을 먼저 먹는다.**

- 군산은 운영·입점 발품의 기준일 뿐, 앱 기본값이나 예시 카피로 노출하지 않는다.
- 사용자 위치는 **저장 지역 → 프로필 활동 지역 → 직접 선택** 순서다.
- 입점·발품은 **군산·근처**부터. 전국 디렉터리는 출시 후
- 관장 경로: **짐 → 체육관 → 관장 입점 관리 → 받은 문의** (별도 관장 앱은 나중)
- 활성화 목표: 군산에 실제 공개 카드가 생기게 하는 것 (빈 목록이어도 “첫 카드가 되세요”로 밀어줌)

## 현재 IA

짐 = **지도 먼저** · 검색 오른쪽 칩 `체육관 · 라이벌` · 왼쪽 레일 `찜` · 하단 시트.
문의·사용자 제보·관장 입점 관리는 체육관 화면의 보조 행동으로 둔다.
