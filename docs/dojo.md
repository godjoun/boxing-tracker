# 도장

> 「이번 주 어디서 스파링하지?」에 답한다.  
> **하단 탭 중앙 옆** (링 · **도장** · 홈 · 명패 · 더보기)  
> 코드: `docs/code-map.md` · 수익: `docs/monetization.md` · 나중에: `docs/later.md`  
> Supabase 한국어 이름표·중간 점검: `docs/supabase.md`  
> 화면 구조(참고 분석): `docs/dojo-ui.md`

| 탭 | 무엇 |
|----|------|
| **모임** | 오픈 스파링 · 합동훈련 (날짜·주소 필수, **검색·날짜 필터**) |
| **체육관 문의·대여** | **찾기 · 내 문의 · 내 관** — 검색→상세→문의·대화 / 관장 등록·받은 문의 |
| **라이벌 찾기** | 공개 카드 · 상호 관심 매칭 · 1:1 대화 |

- 모임 유형을 더 쪼개지 않음
- 연락처 공개 안 함
- 시드 카드 = 예시
- **체육관 검색** — **입점(approved)만** 노출 · 예전 로컬 시드 목록은 검색에서 제외
- **모임 일정·신청** — Supabase 연결 시 폰끼리 동기화 (`supabase/dojo_exchange.sql`) · 미연결이면 로컬
- **체육관 문의** — 리드 저장 · **받은/보낸 문의** · **대화하기**(당근식) · `dojo_inquiry_chat.sql`
- **체육관 입점** — 등록·수정·삭제·사진 · `approved`면 검색 「입점」 · **`is_featured`면 검색 상단 「추천」** · 승인은 Table Editor
- **관 상세** — 목록 → **가게 한 장면** (커버·소개·가격·문의) · 커뮤니티는 한 줄 자리만

- **모임 채팅** — 신청자↔주최자 DM (`supabase/dojo_chat.sql`) · 실시간 없음(새로고침) · 예시 일정 제외
- **라이벌 찾기 v1** — 실제 공개 카드와 관심 요청 (`supabase/dojo_sparring_v1.sql`) · 연락처 공개 없음
- **라이벌 채팅 v2** — 서로 관심을 보낸 뒤에만 1:1 대화 (`supabase/dojo_sparring_chat.sql`) · 4초 폴링 · 푸시 없음
