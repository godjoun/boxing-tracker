# 체육관 추천 노출 2주 파일럿

목적은 결제 기능을 만드는 것이 아니라 체육관의 실제 지불 의사와 유효 문의
증가를 확인하는 것이다.

## 시작 전

- 한 지역에서 운영 중인 체육관 3~5곳을 모집한다.
- 1~2곳만 추천군으로 두고 나머지는 일반 노출 비교군으로 둔다.
- `supabase/gym_featured_pilot.sql`을 적용한다.
- 추천 체육관 행의 `is_featured = true`를 수동 설정한다.
- 노출 기간, 수동 청구액, 중도 중단·환불 조건을 서면으로 확인한다.
- 자동 갱신·장기 계약·앱 결제는 사용하지 않는다.

## 매일 기록할 장부

한 행을 `날짜 × 체육관`으로 두고 아래 값을 기록한다.

- 노출 유형: `featured` / `organic`
- 검색 결과 노출 수: Vercel `gym_search_results_view`
- 상세 진입 수: Vercel `gym_detail_open`
- 문의 열기 수: Vercel `gym_inquiry_open`
- 문의 제출 수: Vercel `gym_inquiry_submit`
- 채팅 열기·전송 수: `gym_inquiry_chat_open`, `gym_inquiry_chat_send`
- 유효 문의 수: 연락 가능하고 체험·대여·예약 의도가 확인된 문의
- 체육관 답변 여부와 실제 방문 여부
- 다음 2주 유료 지속 의사: 예/아니오/금액

이벤트의 `acquisitionSource`가 `featured`인지 확인한다. 문의 DB 집계는
`dojo_featured_pilot_inquiry_metrics` 뷰를 Supabase SQL Editor에서 본다.

## 성공 기준

다음 중 하나를 충족하면 검증 성공으로 본다.

1. 추천 체육관의 상세 진입률이 일반 노출 대비 2배 이상
2. 추천 체육관당 주 1건 이상의 유효 문의
3. 최소 3곳이 다음 기간에도 유료로 지속하겠다고 응답

표본이 너무 작거나 지역·가격이 달라 비교가 왜곡되면 성공으로 간주하지 않고
같은 조건으로 한 번 더 시험한다.

## 종료

- 14일 후 추천 체육관의 `is_featured`를 원복한다.
- 체육관별 지불액, 문의, 방문, 지속 의사를 한 페이지로 정리한다.
- 성공하지 못하면 자동 과금·관장 인증 개발을 시작하지 않는다.
- 성공하면 `docs/post-validation-billing.md`의 순서로만 설계를 시작한다.

## 스타일 프리미엄

기존 6개 스타일 텍스트 드릴과 타이머는 계속 무료다. 새 영상·심화 6~8주
과정은 화면에 잠그지 않고 인터뷰나 별도 대기명단으로만 수요를 확인한다.
