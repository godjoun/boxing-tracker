# 공개 베타 회귀 점검

자동 점검:

```bash
npm test
npm run build
```

`src/utils/coreFlows.test.js`가 스타일 3분 기준·준비시간, 신체 줄넘기
포함/제외·가이드 보존, 타이머 경과 복구, 백업 왕복을 검사한다.

## 기기에서 확인할 흐름

### 일반 타이머 → 자동 기록
- [ ] 홈에서 기본 타이머 시작
- [ ] 준비 10초 → 운동 → 휴식 → 마지막 라운드 완료
- [ ] 완료 화면에서 기록이 한 번만 저장됨
- [ ] 명패/기록에서 라운드와 시간이 맞음
- [ ] 타이머 뒤로가기가 홈으로 복귀

### 4주 코스
- [ ] 링 → 기술 첫 화면이 `4주 코스 = 따라가기`
- [ ] 추천 세션 시작 시 세션별 라운드 시간이 자동 적용
- [ ] 완료 후 해당 세션 진행률 증가
- [ ] 뒤로가기가 4주 코스로 복귀

### 스타일 드릴
- [ ] 기술 → 스타일 → 단계별 드릴 진입
- [ ] 드릴 타이머가 3분 기준·준비 10초로 열림
- [ ] 뒤로가기가 원래 스타일·단계 화면으로 복귀

### 신체
- [ ] 월~토 각 요일에서 줄넘기 포함 시 `워밍업 + 본운동` 라운드
- [ ] 타이머에서 선택한 요일의 운동 목록 표시
- [ ] `줄넘기 제외` 후 본운동 라운드만 남음
- [ ] 앱을 새로 열어도 진행 중 가이드와 현재 라운드 복구
- [ ] 뒤로가기가 신체 요일 화면으로 복귀

### 문의 → 관장 장부 → 채팅
- [ ] 개인정보 동의 전에는 문의 전송 불가
- [ ] 동의 후 문의가 `dojo_gym_inquiries`에 저장
- [ ] 입점 신청 actor와 일치하는 관장 장부에만 노출
- [ ] 문의자·관장 모두 같은 대화방을 열고 메시지 전송
- [ ] 제3의 actor UUID로 방·메시지 RPC 조회 시 0건
- [ ] `beta_rls_hardening.sql` 후 테이블 직접 update/delete/select 차단

### 라이벌 → 상호 관심 → 채팅
- [ ] 한쪽만 관심을 보내면 대화 버튼이 열리지 않음
- [ ] 받은 관심에서 `나도 관심` 후 양쪽 모두 `대화하기` 표시
- [ ] 서로 관심인 두 actor만 같은 대화방을 열고 메시지 전송
- [ ] 관심하지 않은 제3의 actor UUID로 방·메시지 RPC 조회 시 0건
- [ ] 한쪽 공개 카드를 삭제하면 해당 매칭 대화도 제거

### 백업
- [ ] 더보기에서 JSON 내보내기
- [ ] 새 브라우저 프로필에서 JSON 가져오기
- [ ] 기록·명패 복구, 같은 기록 중복 없음
- [ ] 잘못된 JSON·다른 앱 JSON은 오류 안내 후 기존 데이터 유지
