# 도장 화면 구조 (참고 분석 → 우리 반영)

> UI 복제 금지. **흐름·자리만** 빌린다.  
> 제품: `docs/dojo.md` · 수익: `docs/monetization.md`

## 역할 분리 재설계 (보류)

복서용(라이벌·모임·체육관 문의)과 관장용(내 관·받은 문의)을 분리하는
방향은 유지하되, 직전 상단 2단 탭 시안은 만족스럽지 않아 폐기했다.
지금은 기존 3탭과 체육관 내부 역할 탭을 유지하고 나중에 처음부터 다시 설계한다.

## 참고 앱에서 가져온 것

| 앱 | 빌린 구조 | 안 가져온 것 |
|----|-----------|--------------|
| **당근** 비즈프로필 | 찾기 → **가게 홈**(커버·소개·가격) → 문의·채팅 | 단골·쿠폰·후기 피드 |
| **플랩** | 장소/매치 카드 → **상세** → 신청(행동 1개) | 소셜 매치·레벨·구장 예약 풀스택 |
| **Boxing Network** | 짐/이벤트 **허브** + 문의·매칭은 다음 행동 | 팀 가입·bout sheet·코치 워크플로 |
| **더코너류** | (나중) **관에 붙는** 소식·멤버 | 지금 전역 커뮤니티 피드 |

## 관 상세 = 가게 홈 (잠금)

한 장면: **커버 사진 · 관 이름 · 소개 · 위치 · 이용권 · 문의하기(또는 받은 문의)**.  
상세 중에는 상단 역할 탭을 숨긴다. 커뮤니티는 한 줄 자리만.

코드: `GymDetailPanel.jsx`

## 체육관 문의·대여 — 역할 3탭 (현재)

| 탭 | 누구 | 흐름 |
|----|------|------|
| **찾기** | 복서 | 검색 → 관 상세 → 문의하기 → 대화 |
| **내 문의** | 복서 | 보낸 문의 → 미리보기·안 읽음 → 대화하기 |
| **내 관** | 관장 | 등록 · 수정 · 승인 상태 · 받은 문의 |

SQL: `dojo_inquiry_chat.sql` + `dojo_inquiry_chat_inbox.sql`  
푸시·Realtime은 나중.

연락처는 공개하지 않는다. 답장은 앱 대화가 기본.

## 테마

- **라이트:** 흰 무대 · 잉크 텍스트 · 버건디는 CTA만. 스포츠/필름 분위기 없음.
- **다크:** 오일 블랙 + 버건디 틴트 (기존 브랜드감).
